# 1. システム概要

本システムは、Windowsアプリ運用標準に基づいた  
バックグラウンド実行アプリケーションのサンプルである。

タスクスケジューラによる定期実行を前提とし、  
設定・ログ・排他制御を含めた運用構成の実装例を提供する。

---

# 2. 目的

- 開発標準に沿った実装例を提供する  
- タスク実行型アプリの構成を示す  
- 設定・ログ・Mutex・非同期処理の実装例を示す  

---

# 3. 実行頻度

本アプリケーションは以下の実行頻度での運用を想定する。

- 10分ごとに実行  
- 1時間ごとに実行  
- 1日ごとに実行  

※実行頻度はタスクスケジューラにより制御する。

---

# 4. システム構成

本システムは以下で構成される。

- コンソールアプリ（Task Worker）
- 設定ファイル
- ログ出力機能
- タスクスケジューラ
- インストーラ（別途）

---

# 5. モデル定義

挨拶処理の切替を目的として、以下のモデルを定義する。

- 人間クラス（基底クラス）
- 日本人クラス
- 米国人クラス

各クラスは言語に応じた挨拶メッセージを提供する。

---

# 6. 処理モード

処理内容は設定ファイルにより切り替える。

## 6.1 継続実行処理

- 最大10分間処理を継続する  
- 10秒ごとに処理を実行する  
- 開始から1分経過後、設定された言語で挨拶ログを出力する  

---

## 6.2 即時終了処理

- 起動時にメッセージをログ出力し、即時終了する  

---

## 6.3 時間指定終了処理

- 最大10分間処理を継続する  
- 10秒ごとに処理を実行する  
- 開始から1分経過後、挨拶ログを出力する  
- 設定ファイルで指定された時間経過後に処理を終了する  

---

## 6.4 異常終了処理

- 意図的に例外を発生させる  
- エラーログを出力して終了する  

---

# 7. 実行フロー

1. アプリケーション開始ログ出力  
2. 設定ファイル読込  
3. Mutexによる多重起動チェック  
4. 非同期処理開始（Task）  
5. 処理モードに応じた処理実行  
6. 終了ログ出力  
7. アプリケーション終了  

---

# 8. 非同期処理方針

本アプリケーションは、Taskベースの非同期処理を標準とする。

## ■ 基本方針

- Threadの直接利用は行わない  
- Task / async / await を使用する  
- 待機処理は Task.Delay を使用する  

## ■ 停止制御

- CancellationToken を用いて処理停止を制御する  
- 設定時間または処理条件に応じてキャンセルする  

## ■ 実行制御

- 定期処理はループで実装する  
- 実行間隔は設定ファイルで管理可能とする  

## ■ 理由

- 可読性の向上  
- 停止制御の容易化  
- 例外処理の一元化  

---

# 9. パス設計

本アプリケーションでは、設定ファイルの配置先を起点として  
運用用の各種ディレクトリを決定する。

## ■ 基本方針

- 実行ファイルと運用用ファイルは分離する  
- 実行ファイルはインストール先に配置する  
- 設定ファイル、ログ、制御ファイルは ProgramData 配下に配置する  
- インストーラごとに独立した運用領域を持つ  

---

## ■ ディレクトリ構成例

```
C:\ProgramData\Company\TaskWorker10Min\
  ├─ config\
  │  └─ appsettings.json
  ├─ logs\
  └─ control\
     └─ stop_sync.flag
```

---

## ■ ベースディレクトリ決定方法

ベースディレクトリは固定値ではなく、  
起動時に指定された設定ファイルパスから決定する。

例：

```
--config "C:\ProgramData\Company\TaskWorker10Min\config\appsettings.json"
```

この場合のベースディレクトリ：

```
C:\ProgramData\Company\TaskWorker10Min
```

---

## ■ 設定ファイルパスの解決順

1. コマンドライン引数 `--config`  
2. 既定設定（exe と同一ディレクトリ）

---

## ■ インストーラごとの独立性

以下をインストーラ単位で分離する：

- タスク名  
- インストール先  
- 設定ファイル配置先  
- ログ出力先  
- 制御ファイル配置先  

---

# 10. 設定ファイル設計

## ■ 配置場所

```
C:\ProgramData\Company\TaskWorker10Min\config\appsettings.json
```

## ■ 方針

- 設定ファイルはインストーラごとに分離する  
- 起動引数で切り替え可能とする  
- 外部設定を優先して読み込む  

## ■ 設定項目

- ExecutionMode  
- Language  
- TimeoutMinutes  
- MutexKey  
- IntervalSeconds  
- Logging.DirectoryPath  
- Logging.RetentionDays  

---

# 11. ログ仕様

## ■ 出力タイミング

- アプリ開始時  
- アプリ終了時  
- 挨拶実行時  
- 異常発生時  

## ■ ログレベル

- Information：通常ログ  
- Error：異常時  

## ■ 出力先

設定ファイルの Logging.DirectoryPath を優先する。  
未指定の場合は以下を使用する：

```
C:\ProgramData\Company\TaskWorker10Min\logs
```

## ■ ログ保持

- 30日経過したログは削除する  

---

# 12. 設定ファイル

## ■ 配置場所

C:\ProgramData\Company\AppName\config

## ■ 設定項目

- 実行モード（ExecutionMode）
- 言語（Language）
- 処理終了時間（TimeoutMinutes）
- Mutexキー（MutexKey）
- 実行間隔（IntervalSeconds）
- ログ出力先
- ログ保持日数

---

# 13. 終了条件

- 正常終了時は終了コード 0 を返す  
- 異常終了時は 0 以外を返す  

---

# 14. 配布方針

本アプリケーションは、実行頻度ごとにインストーラを分けて配布する。

例：

- 10分実行版
- 1時間実行版
- 1日実行版

各インストーラは同一アプリケーションを使用し、
設定ファイルおよびタスクスケジューラ設定により動作を切り替える。

タスク登録および設定配置はインストーラにより自動化するものとする。

---

# 15. 備考

本仕様書は、開発標準の実装例を示すことを目的とする。  
業務ロジックそのものは対象外とする。

Threadによる実装は可能であるが、本仕様では採用しない。
